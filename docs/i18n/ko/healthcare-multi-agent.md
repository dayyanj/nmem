## 실행 요약

우리는 200명의 환자로부터 생성된 1,705개의 합성 임상 상황을 처리하는 4개의 AI 에이전트(응급처치, 치료, 퇴원, 약국)를 가진 병원 병동의 180일 시뮬레이션을 구축했습니다. 각 에이전트는 nmem의 저널에 기록하고, 시스템은 매일의 통합, 주간의 꿈 상태, 이중 주간의 밤 통합을 실행하여 6개월의 임상 운영을 4.6시간의 벤치마크 실행으로 압축합니다.

이 벤치마크는 인지 메모리가 시간이 지남에 따라 기존의 동일한 LLM이지만 메모리 통합이 없는 기준 대비 에이전트의 답변을 개선하는지 테스트합니다.

### 주요 발견

1. **신념 수정은 nmem의 핵심 기능이다.** 지침 변경 후 nmem은 5.00/5, 기준 3.13/5 (+60%)을 기록하며, 120일 이후 모든 신념 수정 질문에 완벽한 답변을 제공한다.
2. **소비자 하드웨어에서 14B 모델을 사용한 전체적인 개선.** nmem은 205개 평가에서 3.84/5, 기준 3.60/5 (+7%)를 기록하며 단일 RTX 4090에서 Qwen3-14B를 사용한다.
3. **nmem은 77개 질문에서 승리, 47개에서 패배, 81개에서 무승부.** 승리가 신념 수정(+1.54 평균)과 직접 회상(+0.36 평균)에 집중되고, 패배는 시간적 추론(-0.39 평균)에 집중되며, 14B 모델의 제한된 추론 능력이 병목 현상으로 작용한다.
4. **시간이 지남에 따라 격차가 커진다.** 1일: +0.20, 30일: +0.29, 120일: +0.43 — 더 많은 지침 변경과 지식 통합이 이루어질수록 nmem의 우위가 커진다.
5. **모든 기능이 대규모로 실행됨.** 361개의 LTM 승격, 56개의 중복 병합, 1,170개의 충돌 자동 해결, 11개의 꿈 상태 패턴 합성, 6,171개의 지식 링크 — $0의 추론 예산(로컬 vLLM)으로 실행됨.

## 방법론

### 연구 질문

1. 6개월 동안 다중 에이전트 임상 지식을 인지 메모리가 개선하는가?
2. 지침 변경 시 nmem이 신념을 올바르게 수정하는가?
3. 통합 엔진(승격, 중복 제거, 합성)이 측정 가능한 검색 성능을 개선하는가?
4. 이 작업은 소형 인프라(14B 모델, 단일 GPU)에서 실행 가능한가?

### 실험 설계

**데이터 출처:** [Synthea](https://synthetichealth.github.io/synthea/) 합성 환자 생성기 (Apache 2.0). 1,159명의 환자 생성, 200명의 상황 밀도가 높은 환자 선택.

**시뮬레이션 기간:** 180일 (2025-01-01 ~ 2025-06-29)

**에이전트:**

| 에이전트 | 역할 | 처리 |
|---------|------|------|
| triage | 긴급도 평가 | 모든 상황 — 생체 신호, 주요 불편, 우선순위 |
| treatment | 임상 결정 | 진단, 절차, 검사 결과 |
| discharge | 치료 계획 | 치료 계획, 후속 조치, 재입원 추적 |
| pharmacy | 약물 관리 | 처방, 알레르기, 약물 상호작용 |

**주입된 시나리오**(Synthea가 생성할 수 없는 수작업 이벤트):

| 날짜 | 이벤트 | 테스트 기능 |
|------|--------|-------------|
| 30 | 고혈압 임계값 낮춤 140/90 → 130/80 | 신념 수정 |
| 45 | 메트포르민 ER 회수 (NDMA 오염) | 다중 에이전트 전파 |
| 50 | 당뇨병 환자 30일 이내 재입원 | 재입원 패턴 감지 |
| 75 | 당뇨병 2차 치료: sulfonylureas → GLP-1 | 신념 수정 |
| 90 | sulfa 알레르기 환자에게 Bactrim 처방 | 충돌 해결 |
| 110 | 리신오프릴+포타슘 상호작용: 중등도 → 심한 | 신념 수정 |
| 120 | 아편류 초기 처방: 7일 → 3일 | 정책 변경 |
| 130 | 제3 금요일 퇴원 재입원 패턴 | 패턴 감지 |
