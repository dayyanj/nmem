## Spwig 벤치마크: 빠른 수치

**설정:** 17개 저장소의 전자상거래 플랫폼, 45개 테스트 작업, 5개 변형, 225개 이중 판정 평가.

| 변형 | 무엇인가 | 판정 점수 | 비용 |
|---------|-----------|-------------|------|
| **v8_mcp** | 에이전트가 MCP 도구를 통해 nmem을 검색 | 4.27/5 | **$4.35** |
| new_developer | 메모리 없이 처음부터 탐색 | 4.36/5 | $8.18 |
| control | Claude Code 자동 메모리 (82% 사실 커버리지) | 3.98/5 | $7.06 |
| v8_injected | 메모리가 프롬프트에 사전 주입됨 | 4.02/5 | $15.27 |
| v8_briefing | 인식 신호와 함께 briefing API 사용 | 3.96/5 | $18.36 |

**핵심 통찰:** MCP 검색은 에이전트가 각 질문에 따라 무엇을 검색할지 결정하기 때문에 가장 저렴하면서도 정확하다. 사전 주입은 질문을 보지 않고도 유용할 수 있는 것을 추측한다.

## 의료 벤치마크: 빠른 수치

**설정:** 180일 시뮬레이션, 4개 의료 에이전트, 200개 합성 환자, 1,705회 만남, 40개 테스트 질문, RTX 4090에서 Qwen3-14B ($0 추론 비용).

| 범주 | nmem | 기준선 | 차이 |
|----------|------|----------|-------|
| **신념 수정** | **4.75/5** | 3.21/5 | **+48%** |
| 직접 회상 | 4.09/5 | 3.73/5 | +10% |
| 패턴 감지 | 3.48/5 | 3.33/5 | +4% |
| 전체 | 3.84/5 | 3.60/5 | +7% |

**핵심 통찰:** 신념 수정은 nmem의 가장 강력한 차별점이다. 지침이 변경되면 nmem은 모순을 감지하고 통합 단계에서 해결하여 업데이트된 정책을 검색한다. 기준선 LLM은 현재 지식과 오래된 지식을 구분하는 메커니즘이 없다. 120일 후(모든 지침 변경) nmem은 모든 신념 수정 질문에서 완벽한 5.00/5 점수를 기록한다.

## 방법론

### Spwig 벤치마크
- **에이전트:** Claude Sonnet 4.6, 헤드리스 (`claude -p`), 작업당 단일 호출
- **판정자:** Qwen3-14B (GPU) + Qwen3-30B-A3B MoE (CPU), 독립적인 점수, 1-5 척도
- **분리:** 포트 5435에서 실행되는 독립 PostgreSQL, 개발과 분리
- **코퍼스:** 6,076개 항목 — LLM으로 정리된 대화 + 의미적으로 청크화된 자동 메모리 + 문서 + git
- **제어:** new_developer의 경우 CLEAN HOME, 세션 지속 없음, 동일한 CLAUDE.md 기반

### 의료 벤치마크
- **모델:** vLLM에서 Qwen3-14B-AWQ (단일 RTX 4090, $0 추론)
- **판정자:** Qwen3-14B, 척도 1-5에 따라 기준에 따라
- **데이터:** Synthea 합성 환자 (Apache 2.0) + 수작업으로 만든 임상 시나리오
- **시뮬레이션:** 180일을 시간 시뮬레이션을 통해 4.6시간으로 압축
- **평가:** 7개 간격(1, 30, 60, 90, 120, 150, 180일)에 걸쳐 40개 질문
