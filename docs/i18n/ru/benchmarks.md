Эмпирические оценки влияния nmem на производительность агента. Все бенчмарки используют воспроизводимый метод с двойной оценкой судей на локальном оборудовании ($0 стоимость оценки).

## Резюме результатов

| Бенчмарк | Найдено | Ключевой метрика |
|-----------|---------|------------|
| [Spwig Institutional Knowledge](spwig-institutional-knowledge.md) | Поиск nmem MCP соответствует точности нового разработчика с **половинной стоимостью** | 4.27/5 оценка судьи, $0.097/задача |
| [Recognition Signals](recognition-signals.md) | Теги доверия в промптах не меняют поведение моделей 8B-30B | Распознавание вычисляется, но не вставляется |

## Бенчмарк Spwig: Быстрые цифры

**Настройка:** 17-репозиторий платформа электронной коммерции, 45 тестовых задач, 5 вариантов, 225 двойных оценок судей.

| Вариант | Что это | Оценка судьи | Стоимость |
|---------|-----------|-------------|------|
| **v8_mcp** | Агент выполняет поиск в nmem через инструменты MCP | 4.27/5 | **$4.35** |
| new_developer | Нет памяти, исследует с нуля | 4.36/5 | $8.18 |
| control | Автомемория Claude Code (82% охват фактов) | 3.98/5 | $7.06 |
| v8_injected | Память предварительно вставлена в промпт | 4.02/5 | $15.27 |
| v8_briefing | API краткого содержания с сигналами распознавания | 3.96/5 | $18.36 |

**Ключевое наблюдение:** Поиск MCP является самым дешевым и точным, потому что агент решает, что искать, основываясь на каждом вопросе. Предварительная вставка предполагает, что может быть полезно, не видя вопроса.

## Текущий охват

Все бенчмарки до настоящего времени используют **Claude Code (Sonnet 4.6, 200K контекст)** — интеграция MCP является проверенным случаем использования. Агентные случаи с меньшими моделями (8B-30B, 8K-32K контекст) следующий этап в плане. Детали см. в [Области применения и ограничениях](spwig-institutional-knowledge.md#scope--limitations).

## Методология

- **Агент:** Claude Sonnet 4.6, без головы (`claude -p`), одно обращение на задачу
- **Судьи:** Qwen3-14B (GPU) + Qwen3-30B-A3B MoE (CPU), независимая оценка, шкала 1-5
- **Изоляция:** Отдельный PostgreSQL на порту 5435, отдельно от разработки
- **Корпус:** 6 076 записей — разговоры, дистиллированные LLM + семантически разбитая автопередача памяти + документы + git
- **Контроли:** Чистый HOME для new_developer, без сохранения сессии, одинаковый базовый CLAUDE.md
