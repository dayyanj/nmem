nmemがエージェントの性能に与える影響の実証評価。すべてのベンチマークは、ローカルハードウェア（$0の推論コスト）を使用して再現可能な方法論で実施。

## 結果の概要

| ベンチマーク | 発見 | 主要メトリクス |
|-----------|---------|------------|
| [Healthcare Multi-Agent](healthcare-multi-agent.md) | ガイドライン変更後の信念修正スコアは**5.00/5（ベースライン3.13/5）** — 14Bモデル、単一の消費者GPUで実施 | 全体で+7%、信念修正で+60% |
| [Spwig Institutional Knowledge](spwig-institutional-knowledge.md) | nmem MCP検索は、新しい開発者の精度と**半分のコスト**で一致 | 4.27/5の審査スコア、$0.097/タスク |
| [Recognition Signals](recognition-signals.md) | プロンプト内の信頼タグは8B-30Bモデルの挙動に変化を与えない | 認識は計算されるが注入されない |

## Spwigベンチマーク: 簡易数値

**セットアップ:** 17リポジトリのeコマースプラットフォーム、45テストタスク、5バリアント、225の双審査評価。

| バリアント | 内容 | 審査スコア | コスト |
|---------|-----------|-------------|------|
| **v8_mcp** | エージェントがMCPツール経由でnmemを検索 | 4.27/5 | **$4.35** |
| new_developer | メモリなし、スクラッチから探索 | 4.36/5 | $8.18 |
| control | Claude Codeの自動メモリ（82%の事実カバレッジ） | 3.98/5 | $7.06 |
| v8_injected | プロンプトに事前にメモリを注入 | 4.02/5 | $15.27 |
| v8_briefing | 認識シグナル付きのBriefing API | 3.96/5 | $18.36 |

**重要な洞察:** MCP検索は、エージェントが各質問に基づいて検索内容を決定するため、最も安価でありかつ正確である。事前に注入されたメモリは、質問を確認する前に何が役立つかを推測する。

## Healthcareベンチマーク: 簡易数値

**セットアップ:** 180日間のシミュレーション、4つの医療エージェント、200人の合成患者、1,705回のエンカウンタ、40のテスト質問、RTX 4090上のQwen3-14B（$0の推論コスト）。

| カテゴリ | nmem | ベースライン | デルタ |
|----------|------|----------|-------|
| **信念修正** | **4.75/5** | 3.21/5 | **+48%** |
| 直接的な記憶呼び出し | 4.09/5 | 3.73/5 | +10% |
| パターン検出 | 3.48/5 | 3.33/5 | +4% |
| 全体 | 3.84/5 | 3.60/5 | +7% |

**重要な洞察:** 信念修正はnmemの最大の差別化要因である。ガイドラインが変更された場合、nmemは矛盾を検出し、統合時に解決し、更新されたポリシーを取得する。ベースラインLLMには、現在の知識と古い知識を区別するメカニズムがない。120日目（すべてのガイドラインが変更された後）では、nmemはすべての信念修正質問で完璧な5.00/5を獲得した。

## 方法論

### Spwigベンチマーク
- **エージェント:** Claude Sonnet 4.6、ヘッドレス（`claude -p`）、タスクごとに単一の呼び出し
- **審査者:** Qwen3-14B（GPU）+ Qwen3-30B-A3B MoE（CPU）、独立したスコアリング、1-5スケール
- **分離:** ポート5435の専用PostgreSQL、開発環境とは別
- **コーパス:** 6,076エントリ — LLMで精錬された会話 + 論理的にチャンク化された自動メモリ + ドキュメント + git
- **制御:** new_developerのためのクリーンなHOME、セッションの永続化なし、同じCLAUD.mdベース

### Healthcareベンチマーク
- **モデル:** vLLM上のQwen3-14B-AWQ（単一のRTX 4090、$0の推論）
- **審査者:** Qwen3-14B、基準に従った1-5スケール
- **データ:** Synthea合成患者（Apache 2.0）+ 手作業で作成された臨床シナリオ
- **シミュレーション:** 180日を時間シミュレーションで4.6時間に圧縮
- **評価:** 7つの間隔（日1、30、60、90、120、150、180）で40の質問
