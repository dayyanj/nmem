## まとめ

我々は、4つのAIエージェント（トリアージ、治療、退院、薬局）が200人の患者から生成された1,705件の合成臨床エンカウンタを処理する180日間の病棟シミュレーションを構築しました。各エージェントはnmemのジャーナルに書き込み、システムは毎日のコンソリデーション、週次のドリームステート、および2週間ごとのナイトリースンセシスを実行します。これにより、6か月分の臨床運用を4.6時間のベンチマーク実行に圧縮します。

このベンチマークは、同じLLMを使用したベースラインと比較して、認知メモリがエージェントの回答を時間とともに改善するかをテストします。

### 主な発見

1. **信念の修正はnmemの最大の特徴です。** ガイドラインの変更後、nmemは5.00/5、ベースラインは3.13/5（+60%）を記録し、120日以降のすべての信念修正質問に完璧な回答を提供します。
2. **消費者向けハードウェアで14Bモデルを使用した全体的な改善。** nmemは205回の評価で3.84/5、ベースラインは3.60/5（+7%）を記録し、単一のRTX 4090でQwen3-14Bを使用しています。
3. **nmemは77問を勝ち、47問を負け、81問を引き分けました。** 勝利は信念修正（+1.54平均）と直接的な記憶（+0.36平均）に集中しています。敗北は時間的推論（-0.39平均）に集中しており、14Bモデルの限られた推論能力がボトルネックである可能性があります。
4. **時間とともに差は広がります。** 1日目：+0.20、30日目：+0.29、120日目：+0.43 — より多くのガイドラインが変更され、知識がコンソリデーションされるにつれて、nmemの優位性が増加します。
5. **すべての機能がスケールで実行されます。** 361件のLTMプロモーション、56件の重複マージ、1,170件の衝突自動解決、11件のドリームステートパターンの合成、6,171件の知識リンク — 0ドルの推論予算（ローカルvLLM）で。

## 方法論

### 研究質問

1. 6か月間の多エージェント臨床知識は認知メモリによって改善されるか？
2. nmemはガイドラインの変更時に信念を正しく修正できるか？
3. コンソリデーションエンジン（プロモーション、重複、合成）は測定可能な検索改善をもたらすか？
4. この作業は、中規模のインフラ（14Bモデル、単一GPU）で実行可能か？

### 実験設計

**データソース：** [Synthea](https://synthetichealth.github.io/synthea/)（Apache 2.0ライセンス）による合成患者生成。1,159人の患者が生成され、エンカウンタ密度の上位200人がシミュレーションウィンドウに選ばれました。

**シミュレーションウィンドウ：** 180日（2025-01-01から2025-06-29）

**エージェント：**

| エージェント | 役割 | 処理内容 |
|------------|------|----------|
| triage | 緊急性評価 | すべてのエンカウンタ — 生体情報、主訴、優先度 |
| treatment | 臨床的決定 | 診断、手術、検査結果 |
| discharge | 転院の調整 | 転院計画、フォローアップ、再入院の追跡 |
| pharmacy | 薬物管理 | 薬の処方、アレルギー、薬物相互作用 |

**注入されたシナリオ**（Syntheaが生成できない手作業のイベント）：

| 日 | イベント | テストされた機能 |
|----|---------|----------------|
| 30 | 高血圧の閾値が140/90 → 130/80に低下 | 信念の修正 |
| 45 | メトホルミンERのリコール（NDMA汚染） | クロスエージェントの伝播 |
| 50 | 糖尿病患者が30日以内に再入院 | 再入院パターンの検出 |
| 75 | 糖尿病2番目の治療：スルホニルウレア → GLP-1 | 信念の修正 |
| 90 | サルファアレルギーを持つ患者にバクトラム処方 | 衝突解決 |
| 110 | リシノプリル+カリウムの相互作用：MODERATE → SEVERE | 信念の修正 |
| 120 | オピオイドの初期処方：7日 → 3日 | 政策変更 |
| 130 | 第3金曜日退院の再入院パターン | パターン検出 |

**日常のシミュレーションサイクル：**
