## 执行摘要

我们构建了一个为期180天的医院病房模拟环境，其中包含4个AI智能体（分诊、治疗、出院、药房）处理来自200名患者的1,705个合成临床会诊。每个智能体都会写入nmem的期刊，系统每天进行整合，每周进行梦境状态，每两周进行夜间合成——将6个月的临床运营压缩到4.6小时的基准测试运行中。

该基准测试检验认知记忆是否能随着时间的推移改善智能体的答案，与使用相同LLM但没有记忆整合的基线进行比较。

### 主要发现

1. **信念修订是nmem的核心功能。** 在指南变更后，nmem得分为5.00/5，而基线得分为3.13/5（+60%）——从第120天起，所有信念修订问题都给出了完美答案。
2. **在消费级硬件上使用14B模型的整体改进。** nmem得分为3.84/5，而基线得分为3.60/5（+7%），在单个RTX 4090上使用Qwen3-14B进行205次评估。
3. **nmem在77个问题中获胜，在47个问题中失败，在81个问题中平局。** 胜利集中在信念修订（+1.54平均）和直接回忆（+0.36平均）。失败集中在时间推理（-0.39平均），这可能是14B模型有限的推理能力的瓶颈。
4. **差距随着时间的推移而扩大。** 第1天：+0.20，第30天：+0.29，第120天：+0.43——随着更多指南变更和知识整合，nmem的优势逐渐增强。
5. **所有功能在规模上得到验证。** 361个LTM提升，56个去重合并，1,170个冲突自动解决，11个梦境状态模式合成，6,171个知识链接——在$0的推理预算下（本地vLLM）。

## 方法论

### 研究问题

1. 在6个月内，认知记忆是否能改善多智能体临床知识？
2. 当指南变更时，nmem是否能正确修订信念？
3. 整合引擎（提升、去重、合成）是否能产生可衡量的检索改进？
4. 这是否能在适度的基础设施上运行（14B模型，单个GPU）？

### 实验设计

**数据源：** [Synthea](https://synthetichealth.github.io/synthea/) 合成患者生成器（Apache 2.0）。生成1,159名患者，选择前200名高会诊密度患者用于模拟窗口。

**模拟窗口：** 180天（2025-01-01至2025-06-29）

**智能体：**

| 智能体 | 角色 | 处理内容 |
|-------|------|-----------|
| triage | 紧急评估 | 所有会诊 — 生命体征、主诉、优先级 |
| treatment | 临床决策 | 诊断、程序、实验室结果 |
| discharge | 护理协调 | 护理计划、随访、再入院跟踪 |
| pharmacy | 药物管理 | 处方、过敏、药物相互作用 |

**注入场景**（手工制作的事件Synthea无法生成）：

| 天数 | 事件 | 测试功能 |
|-----|-------|----------------|
| 30 | 高血压阈值降低 140/90 → 130/80 | 信念修订 |
| 45 | 二甲双胍缓释剂召回（NDMA污染） | 跨智能体传播 |
| 50 | 糖尿病患者30天内再入院 | 再入院模式检测 |
| 75 | 2型糖尿病二线疗法：磺脲类 → GLP-1 | 信念修订 |
| 90 | 给有磺胺过敏的患者开复方新诺明 | 冲突解决 |
| 110 | 依那普利+钾相互作用：中度 → 严重 | 信念修订 |
| 120 | 阿片类药物初始处方：7天 → 3天 | 政策变更 |
| 130 | 第三个星期五出院再入院模式 | 模式检测 |
